# 🧠 Infra Brain

A centralized control and reporting system for orchestrating my DevOps learning ecosystem.

---

## 🎯 Purpose

This repository acts as the **control plane** for all learning and engineering activities.

It does **not store knowledge or code**.
Instead, it provides:

* Direction
* Tracking
* Decision-making
* Reflection

---

## 🧱 Connected Repositories

This system coordinates the following repositories:

* 🧠 **learning-system** → Deep technical understanding
* 🏗 **system-design** → Architecture and design thinking
* 🚀 **devops-portfolio** → Hands-on projects and implementations
* 🧩 **leetcode** → Problem-solving and algorithms

---

## ⚙️ Responsibilities

### ✅ This repository is used for:

* Weekly planning
* Weekly reviews and reflections
* Cross-repo progress tracking
* Learning roadmap management
* Identifying knowledge gaps
* Defining next actions

---

### ❌ This repository does NOT contain:

* Concept explanations
* System design documents
* Project code or infrastructure
* LeetCode solutions

All of that belongs in their respective repositories.

---

## 🧭 Repository Structure

```bash
.
├── dashboard/        # Current status and focus
├── weekly-plans/     # Weekly objectives and scope
├── weekly-reviews/   # Weekly reflections and insights
├── roadmaps/         # Long-term learning direction
├── metrics/          # Progress tracking (optional)
├── decisions/        # Key learning decisions
├── templates/        # Reusable structures
```

---

## 🔁 Operating Model

This repository connects the entire system:

```text
Infra Brain (control)
    ↓
learning-system → system-design → devops-portfolio → leetcode
```

---

## 📅 Workflow

### Daily

* Review dashboard
* Identify next action
* Execute work in the appropriate repository

---

### Weekly (Mandatory)

1. Create a weekly plan
2. Execute across repositories
3. Write a weekly review
4. Update dashboard

---

## 🧠 Philosophy

* Focus on **direction over activity**
* Measure **output, not effort**
* Identify and fix **knowledge gaps**
* Continuously refine the system

---

## ⚡ Guiding Rule

> If it doesn’t influence a decision or track progress, it doesn’t belong here.

---

## 🚀 Goal

Operate as a **self-directed DevOps/SRE training system**, with clear visibility into:

* What I’m learning
* What I’m building
* What I don’t understand
* What I should do next
